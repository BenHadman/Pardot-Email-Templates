A few words before diving in

Framework

The Blacktrace brand email templates are built using the Acorn Email Framework.

HTML Email has complex compatibility issues with the large variety of email clients available rendering html in different ways, by using a framework such as Acorn we can ensure that the code has been tested across many clients and is robust as possible.

The templates should be considered to have modular blocks which can be reconfigured quickly to build an email for whatever needs you have. It is not intended for these blocks to be adjusted in structure or styling from their existing form.

Design Philosophy

The templates have been designed in such a way so that the product marketer simply needs to arrange images and write copy and doesn't spend too long composing and testing new designs or half-baked designs from previous campaigns.

It can be easy to get carried away with fixing every tiny aspect of an email, remember to bear in mind that once you factor in the open rate and then consider the even smaller proportion that take the time to read the email your audience is very small. Perhaps spending time on a more impactful part of you campaign like the copy would be a better use of your time.

Done is better than perfect

The templates should be considered a constantly evolving piece of work with minor bug fixes corrected as we go along.

Each brand has a main template held within Pardot, the template is a large series

Pardot Regions

Pardot email templates adds control to the html template for the frontend visual editor to read correctly and make it easy to edit content. It does this by adding an attribute to the html specifying which type of editable region it is, they are highlighted in green and red here: 

<a href="https://blacktrace.com" pardot-region="editable-link"pardot-region-type="link">My Link</a>
There are four types of editable region which each have different levels of editing freedom.

HTML

Full html control of the region.

<td pardot-region=""/<td>

Image

Allows the control of image dimensions, link, source and alt text.

<img src="image/my-image.png" pardot-region="editable-image" pardot-region-type="image" />

Simple

Only allows control of text.

<p pardot-region="simple-text" pardot-region-type="simple">Your text here</p>

Link

Allows control of text and ahref link.

<a href="https://blacktrace.com" pardot-region="editable-link" pardot-region-type="link">My Link</a>

Throughout the template, we use #EEEEEE for the 'outside' colour (basically, the email body), and #FFFFFF or #232323 for the inside of a section. Spacer - Boxed

The Boxed Spacer is basically a section with just a spacer inside the container. Being limited to the container's 600px width, you can use different background colours on the full width class="wrapper" and on the class="container".

Spacer - Full Width

Same as the Boxed one, only that the wrapper doesn't have a container inside - it just houses the Spacer directly, so you can use it to create vertical space in the same colour as the email body.

Divider - Boxed

A Divider component wrapped inside a container, in the same way the Boxed Spacer is.

Divider - Full Width

A full width divider that will stretch from edge to edge in your email client's window or preview pane.

Limitations & Other Notes

Hamburger menu client support

The interactive hamburger menu has limited support on mobile, because it needs CSS :hover and this isn't supported everywhere. The good news is that Apple Mail and Outlook support it, and the former is one of the most used mobile email clients.

As far as Gmail on mobile goes, Google have removed support for :hover. Our template targets Gmail apps directly and makes the hamburger menu fall back to a basic menu instead, so your subscribers can actually use the menu.

Email builders and menu items

Because navigation menus for email can be coded in so many different ways, currently no email builder provides an easy way to add extra menu items (or sometimes even remove them). Such customisations will need to be done in code.

Utilities for custom mobile email spacing

Acorn includes padding-based responsive utility classes for fine-tuned control over spacing on mobile. Following Bootstrap 4 naming conventions, the classes are named using the format p{sides}-sm-{size}:

p stands for 'padding', use as-is.

{sides} is one of:

  • t - for classes that set padding-top
  • r - for classes that set padding-right
  • b - for classes that set padding-bottom
  • l - for classes that set padding-left
  • x - for classes that set both padding-left and padding-right
  • y - for classes that set both padding-top and padding-bottom
  • blank - for classes that set padding on all 4 sides of the element

{size} is one of:

  • 0 - for classes that eliminate padding by setting it to 0
  • 8 - for classes that set padding to one fourth of container apparent padding (8px default)
  • 16 - for classes that set padding equal to half of container apparent padding (16px default)
  • 24 - for classes that set padding equal to half of container apparent padding (24px default)
